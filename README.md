# 武汉二手房数据分析项目

这是一个独立创建的 Python 数据分析项目，面向武汉二手房挂牌数据，完成了从原始数据清洗、标准化、统计分析，到词云、地图可视化和聚类分析的一整套流程。

项目的目标不是只做单张图表，而是把一份公开数据整理成一个可以直接运行、可以复现结果、也可以作为 GitHub 展示项目的完整分析工程。它强调的是数据流的完整性、代码结构的清晰性，以及输出结果的可读性。

## 项目概述

本项目围绕武汉二手房挂牌数据展开，主要完成以下几件事：

1. 读取公开数据源中的原始 CSV 文件。
2. 将原始字段整理为统一的标准化数据。
3. 对价格、面积、户型、用途等维度做基础统计分析。
4. 生成词云和多种可视化图表，用于观察数据分布和特征。
5. 在配置百度地图 AK 后，补充地理编码、地图展示和聚类分析。
6. 将中间结果和最终图表分别写入固定目录，方便复用和检查。

这意味着仓库本身不仅包含分析脚本，也包含完整的数据处理路径：原始数据、标准化数据、分析脚本、资源文件和输出结果目录都能对应起来。

## 核心功能

- 数据清洗与标准化
- 房源价格、面积、户型、用途等基础分析
- 词云生成
- 地图可视化
- 聚类分析
- 统一主入口脚本，一键串联整个流程

## 数据来源

本项目使用的原始 CSV 数据来自武汉市数据开放平台：

- [武汉二手房挂牌信息内容](https://data.wuhan.gov.cn/page/data/data_set_details.html?cataId=8954e103818545d1877e5f2b99824b6bc)

原始数据文件在仓库中以 [二手房挂牌信息内容_2026-05-04.csv](二手房挂牌信息内容_2026-05-04.csv) 的形式保留，作为项目输入数据使用。

## 项目结构

- `数据分析程序/data_analysis/main.py`：主入口，按顺序调度清洗、分析、词云、地图和聚类
- `数据分析程序/data_analysis/analysis_common.py`：公共配置、数据加载和标准化导出
- `数据分析程序/data_analysis/preprocessing/`：原始数据清洗与标准化
- `数据分析程序/data_analysis/analysis/`：价格、面积、户型、用途、词云和地图等分析脚本
- `数据分析程序/data_analysis/clustering/`：聚类分析与结果输出
- `数据分析程序/data_analysis/assets/`：字体、背景图等静态资源
- `数据分析程序/data_analysis/artifacts/`：运行时生成的标准化数据和中间文件
- `数据分析程序/data_analysis/目录说明.md`：目录关系说明
- `requirements.txt`：依赖列表

## 数据流说明

整个项目的数据流比较清晰，基本可以概括为下面这条链路：

原始 CSV 数据 -> 标准化数据 -> 各类分析图表 -> 地图中间文件 -> 聚类结果

其中：

- `preprocessing/test_clean.py` 负责把原始数据整理成统一格式。
- `analysis_common.py` 提供数据读取、字段适配和公共配置。
- `analysis/*.py` 负责不同维度的统计分析和可视化。
- `analysis/ciyun.py` 负责词云生成。
- `analysis/baidu_map.py` 负责地理编码并输出坐标文件。
- `clustering/run.py` 读取坐标文件并执行聚类分析。

## 运行方式

### 环境要求

- Python 3.9 或更高版本
- 建议使用虚拟环境

### 安装依赖

```bash
pip install -r requirements.txt
```

### 启动项目

进入项目根目录后，执行：

```bash
cd 数据分析程序/data_analysis
python main.py
```

如果你使用的是虚拟环境，先激活虚拟环境再执行上述命令。

## 运行流程

主入口会按以下顺序执行：

1. 读取武汉二手房原始 CSV 数据。
2. 生成标准化数据文件。
3. 执行价格、面积、户型、用途等统计分析。
4. 生成词云和其他图片结果。
5. 如果配置了百度地图 AK，则执行地理编码和地图分析。
6. 如果坐标文件可用，则继续执行聚类分析并输出结果。

## 地图与聚类配置

地图和聚类属于可选增强功能，依赖百度地图开放平台：

- 需要设置环境变量 `BAIDU_MAP_AK`
- 需要在百度地图控制台把当前机器公网 IP 加入白名单

如果没有配置 AK 或白名单，程序会自动跳过地图和聚类步骤，但前面的清洗和常规分析仍然可以正常运行。

## 输出结果

运行后生成的文件会分别保存在以下目录：

- `数据分析程序/data_analysis/artifacts/`
- `数据分析程序/data_analysis/analysis/picture/`
- `数据分析程序/data_analysis/clustering/result/`

这些目录里包含标准化数据、坐标中间文件、图片结果和聚类输出，属于运行产物，一般不需要提交到 GitHub。
